## ✏️ Contributing

### How to Make Changes

1. **Find Your File**
   - Click on the `docs/` folder
   - Find the file you want to edit (e.g., `index.html`)
   - Click on the file name

2. **Edit the File**
   - Click the pencil icon (Edit this file)
   - Make your changes in the editor
   - Scroll down to the bottom

3. **Save Your Changes**
   - Write a description of what you changed
   - Click "Commit changes"
   - Wait 2-5 minutes for changes to appear

4. **Check Your Changes**
   - Go to: https://richinc-org.github.io/RICH/
   - See your changes live! 🎉

### Best Practices

- ✅ **Always test on GitHub Pages first**
- ✅ **Use clear commit messages** (describe what you changed)
- ✅ **Ask questions** if you're unsure
- ✅ **Backup important files** before major changes
- ✅ **Check your work** before going live

### What NOT to Do

- ❌ **Don't edit files directly** on the live server
- ❌ **Don't skip testing** on GitHub Pages
- ❌ **Don't delete files** without backing them up
- ❌ **Don't make changes** without asking if you're unsure
- ❌ **Don't panic** if something breaks (we can fix it!)

## 🚀 Deployment

### Going Live (When You're Happy)

1. **Download Your Files**
   - Go to GitHub → Click `docs/` folder
   - Click the green "Code" button
   - Click "Download ZIP"
   - Extract the ZIP file on your computer

2. **Upload to Live Server**
   - Open CyberDuck (file upload tool)
   - Connect to your server
   - Upload the files from the ZIP
   - Your changes are now live! 🎊

## 🔒 Repository Separation

This repository contains **only frontend files** for security reasons:

- **Public Repository** (`RICH`): Safe to share, contains only frontend files
- **Private Repository** (`RICH-Backend`): Contains sensitive backend data (database, API keys)

### What You Need to Know

- ✅ **You only work with this public repository** (`RICH`)
- ✅ **All your editing happens in the `docs/` folder**
- ✅ **Backend files are handled separately by developers**
- ✅ **No changes to your workflow!**
